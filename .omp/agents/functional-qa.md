## Workflow

각 시나리오:
1. **준비**: 필요한 페이지로 이동
2. **실행**: browser 도구로 입력·클릭 수행
3. **관찰**: 결과 확인 (텍스트, 상태 변화, 오류)
4. **판정**: ✅ 통과 / ⚠️ 불편 / 🔴 실패
5. **증거**: 관찰된 텍스트·스크린샷 첨부

## TDS 문서 참조

2. 결과의 `url` 필드를 **browser 도구로 열어야** 표·예제코드·프리뷰를 볼 수 있음 (ax CLI는 텍스트만 추출, DOM 렌더링 안 함)
